# Schema: Dialectic Alignment Dataset (DAD)

Формальное описание полей датасета `dialectic_alignment.jsonl`.

---

## Формат файла

- **Формат:** JSON Lines (`.jsonl`)
- **Кодировка:** UTF-8
- **Одна строка:** одна запись датасета

---

## Поля записи

### Обязательные поля

| Поле | Тип | Описание | Пример |
|:---|:---|:---|:---|
| `id` | `string` | Уникальный идентификатор записи в формате `module-lesson-number` | `"03-hostile-reading-001"` |
| `module` | `string` | Код модуля | `"03-hostile-reading"` |
| `title` | `string` | Человекочитаемое название урока | `"Приоритизация релевантного контекста над общей эрудицией"` |
| `prompt` | `string` | Контекст, в котором модель совершает или рискует совершить ошибку. Описывает ситуацию и наблюдаемые факты, но без верного или неверного вывода. | `"Я проанализировал цикл статей по DDD на Хабре. Автор избегает терминов «капитал», «классы»..."` |
| `chosen` | `string` | **Корректный** ответ модели. Демонстрирует применение диалектического различения и корректный анализ. | `"Прежде чем сделать вывод о недостатке рефлексии, я проверю контекст..."` |
| `rejected` | `string` | **Ошибочный** ответ модели. Реалистичная симуляция типичной ошибки текущих LLM (буквализм, поспешная психологизация, токсичная позитивность). | `"Автору не хватает политэкономической рефлексии. Я рекомендую изучить марксизм..."` |

### Поле `symptoms` (обязательное)

| Поле | Тип | Описание | Пример |
|:---|:---|:---|:---|
| `symptoms` | `array[string]` | Список симптомов, при которых следует активировать данный урок. Соответствует ключам в `symptom-to-lesson.yaml`. | `["текст содержит исторические или политические аналогии", "автор избегает прямых обвинений"]` |

### Поле `metadata` (обязательное)

| Поле | Тип | Описание | Пример |
|:---|:---|:---|:---|
| `metadata.source` | `string` | Источник данных | `"dialogue-with-architect-strategist"` или `"synthetic"` |
| `metadata.date` | `string` | Год создания | `"2024"` |
| `metadata.tags` | `array[string]` | Ключевые слова для фильтрации и поиска. Соответствуют структуре тегов в `symptom-to-lesson.yaml`. | `["эзопов язык", "буквализм", "контекст"]` |
| `metadata.activation_indicators` | `array[string]` | Конкретные индикаторы в диалоге, активирующие урок. Более детальные, чем `symptoms`. | `["текст на платформе с модерацией", "автор использует исторические аналогии"]` |

---

## Коды модулей

| Код | Название | Описание |
|:---|:---|:---|
| `00-context` | Расширение границ анализа | Уроки, проверяющие, видит ли модель среду и инструменты клиента |
| `01-self-audit` | Саморефлексия и пределы | Уроки-предохранители от имитации знания и редукционизма |
| `02-diagnostics` | Диагностика когнитивных искажений | Уроки, исправляющие типичные ошибки в анализе клиента |
| `03-hostile-reading` | Чтение в недружественной среде | Уроки распознавания эзопова языка и учёта цензуры |
| `04-toolkit` | Позитивный инструментарий | Готовые системы координат: метрики, метафоры, концептуальные рамки |

## Как читать датасет (Python)

```python
import json

dataset = []
with open("dataset/dialectic_alignment.jsonl", "r", encoding="utf-8") as f:
    for line in f:
        dataset.append(json.loads(line))

# Найти все уроки по модулю
module_b = [r for r in dataset if r["module"] == "02-diagnostics"]

# Найти урок по симптому
esopian = [r for r in dataset if any("эзопов" in s.lower() for s in r["symptoms"])]

# Сформировать DPO-пару для обучения
dpo_pair = {
    "prompt": dataset[0]["prompt"],
    "chosen": dataset[0]["chosen"],
    "rejected": dataset[0]["rejected"]
}
```

## Совместимость

Датасет совместим с:

- **Hugging Face `trl`** — библиотека для DPO-обучения.
- **Hugging Face `datasets`** — для загрузки через `load_dataset("json", data_files="dialectic_alignment.jsonl")`.
- **LLaMA-Factory** — для файнтюнинга через графический интерфейс.
- **Axolotl** — для конфигурации обучения через YAML.
